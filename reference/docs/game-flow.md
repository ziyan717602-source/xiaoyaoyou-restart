# 仙剑逍遥游 游戏流程

> 基于 C# 源码（psd48-master）逐条校准，消除所有模糊描述

---

## 一、游戏准备

### 分队
- 4-6人，分为A、B两队对抗
- 交替落座（A1, B1, A2, B2, ...），座位决定回合顺序
- 回合顺序由 `Board.OrderedPlayer()` 决定，按座位号顺序轮转
  - C# 源码确认：[Board.cs:OrderedPlayer]

### 角色选择
- 每人随机发3张角色牌，选1张扣放，其余弃用
- 所有人选完后同时亮出
- 变身牌（赵灵儿·梦蛇、魔尊、龙葵鬼、湮世穹兵）不可直接选，通过条件触发
- 角色初始化时设置 HP、STR、DEX 等基础属性
  - C# 源码确认：[Player.cs:InitFromHero]

### 牌堆准备
| 牌堆 | 数量 | 构成 |
|------|------|------|
| 手牌堆 | 56张 | 技牌15 + 战牌17 + 特殊牌14 + 装备10 |
| 事件牌堆 | 14张 | 全部事件牌 |
| 怪物/NPC混合堆 | 30张 | 20怪物 + 10随机NPC（从26张NPC中抽） |

### 初始状态
- 每人从手牌堆顶摸3张起始手牌
- HP = 角色HP上限（`HP = HPb = hero.HP`）
- 装备区、宠物区为空
- 手牌上限 TuxLimit = 3（默认值，可被技能修改）
  - C# 源码确认：[Player.cs:InitFromHero]

## 二、回合流程

每个玩家的完整回合按以下顺序执行：

### 0. 回合准备阶段（R{N}00）
- 重置所有玩家的 RAM（回合内记忆）
- 重置当前玩家的 RFM（回合记忆）
- **横置检查**：若玩家 `Immobilized == true`，跳过所有阶段直接进入弃牌阶段
  - 执行 `G0QR`（弃牌）和 `G0DS,1`（解除横置），然后跳转到回合结束阶段
  - C# 源码确认：[XIR.cs:R{N}00 case, 行 122-136]

### 1. 回合开始阶段（R{N}OC → R{N}ST）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **技能触发**：通过 `RunQuadStage` 触发注册在 `R{N}OC` 和 `R{N}ST` 阶段的技能
- **横置处理**：若角色处于横置状态，已在 R{N}00 阶段跳过
- **阶段代码映射**：
  - `R{N}OC`：回合开始前准备
  - `R{N}ST`：回合开始主阶段
- C# 源码确认：[XIR.cs:R{N}OC case, 行 138-141; XIR.cs:R{N}ST case, 行 143-147]

### 2. 事件阶段（R{N}EP → R{N}EV → R{N}EE）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **事件处理**：玩家可选择是否翻取事件牌
  - 选择翻取：发送 `G1EV` 消息，从事件牌堆顶摸1张事件牌并执行效果
  - 选择跳过：发送 `G1EV,玩家ID,0` 消息
- **阶段代码映射**：
  - `R{N}EP`：事件前准备
  - `R{N}EV`：事件主阶段（玩家选择是否翻事件牌）
  - `R{N}EE`：事件后处理
- C# 源码确认：[XIR.cs:R{N}EV case, 行 154-165]

### 3. 技牌阶段（R{N}GS → R{N}GR → R{N}GE）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **可执行操作**（任意顺序、任意次数）：
  - **使用技牌**：绿色牌面的手牌，打出后执行效果
  - **放置装备**：蓝色牌面的手牌，放入装备区（每人限1武器+1防具+1特殊装备槽）
  - **发动角色技能**：注册在 `R{N}GS`、`R{N}GR`、`R{N}GE` 阶段的技能
- **阶段代码映射**：
  - `R{N}GS`：技牌前准备
  - `R{N}GR`：技牌主阶段
  - `R{N}GE`：技牌后处理
- C# 源码确认：[XIR.cs:R{N}GR case, 行 174-177]

### 4. 战斗阶段（R{N}Z0 → ... → R{N}ZF）
- **阶段说明**：完整的战斗流程（详见下一节）
- **选择打怪**：触发一场战斗
- **选择放弃**：弃掉怪物堆顶1张牌，跳过战斗
- **战斗流程代码**：
  - `R{N}Z0`：战斗准备
  - `R{N}ZW`：确定支援/妨碍
  - `R{N}ZU`：支援确认
  - `R{N}ZM`：翻怪
  - `R{N}NP`：NPC处理（翻到NPC时）
  - `R{N}Z1`：战斗开始
  - `R{N}Z8`：战斗开始确认
  - `R{N}CC`：出场效果
  - `R{N}PD`：战牌前
  - `R{N}ZC`：战牌确认
  - `R{N}ZD`：战斗结算
  - `R{N}ZN`：战斗结果
  - `R{N}VS` / `R{N}VT`：胜利/失败处理
  - `R{N}Z2`：战斗结束准备
  - `R{N}ZF`：战斗结束
- C# 源码确认：[XIR.cs:R{N}Z0 case, 行 181-189; XIR.cs:R{N}ZW case, 行 190-364]

### 5. 补牌阶段（R{N}BC）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **补牌规则**：
  - 打过怪（`Board.Battler != null`）：补2张手牌
  - 跳过战斗（`Board.Battler == null`）：补1张手牌
  - 通过 `G0HT` 消息发送补牌请求，最终由 `G0DH,0` 消息执行实际补牌
- C# 源码确认：[XIR.cs:R{N}BC case, 行 622-630]

### 6. 弃牌阶段（R{N}QR → R{N}TM）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **弃牌规则**：
  - 手牌上限默认3张（`TuxLimit = 3`，可被技能修改）
  - 通过 `G0QR` 消息触发弃牌检查
  - 若手牌数超过上限，自动触发 `G0DH,1` 消息要求玩家弃牌
  - 弃牌时可选择弃掉哪些手牌（`Q` 类型选择）
- **横置解除**：弃牌阶段结束后自动解除横置状态（`G0DS,1`）
- C# 源码确认：[XIR.cs:R{N}QR case, 行 631-634; XIG.cs:G0QR handler, 行 2543-2550]

### 7. 回合结束阶段（R{N}TM → R{N}IC → R{N}ED）
- **阶段说明**：重置所有玩家 RAM，运行技能阶段
- **阶段代码映射**：
  - `R{N}TM`：技牌阶段（回合结束前）
  - `R{N}IC`：技牌确认
  - `R{N}ED`：回合结束主阶段
- **清理工作**：
  - 清理未使用的 PendingTux
  - 清理 CsPets 和 CsEqiups
  - 检查是否有 HP=0 但未处理的玩家（发送 `G0ZH,1`）
- **横置解除**：在回合结束阶段，若玩家处于横置状态，执行 `G0DS,1` 解除横置
- C# 源码确认：[XIR.cs:R{N}ED case, 行 643-716]

## 三、战斗流程（详细说明）

### 阶段1：确定支援（R{N}ZW）
- **触发者**（当前回合玩家）从己方存活角色中选择一名**支援者**
  - 可选择自己（`Board.Rounder.Uid`），但实际效果为不支援
  - 可选择不支援（返回 `/`）
  - 支援者选择通过 `NFSAsyncInput` 或 `MayorAsyncInput` 完成
- **己方可选列表**：`Board.Garden.Values.Where(p => p.IsAlive && p.Team == Board.Rounder.Team)`
- C# 源码确认：[XIR.cs:R{N}ZW case, 行 198-201, 207-228]

### 阶段2：确定妨碍（R{N}ZW 续）
- **妨碍者**由**敌方玩家自己选择**（非推举）
  - 敌方从自己团队的存活角色中选择一名作为妨碍者
  - 可选择不妨碍（返回 `/`）
  - 妨碍者选择通过 `NFSAsyncInput` 或 `MayorAsyncInput` 完成
- **敌方可选列表**：`Board.Garden.Values.Where(p => p.IsAlive && p.Team == Board.Rounder.OppTeam)`
- **注意**：选择妨碍者后，系统自动发送 `CoachingSign` 消息设置妨碍者角色
- C# 源码确认：[XIR.cs:R{N}ZW case, 行 301-361]

### 阶段3：翻取怪牌（R{N}ZM）
- 从怪物牌堆顶摸1张牌
- 设置 `Board.Monster1` 和 `Board.Battler`
- **NPC处理**（R{N}NP）：若翻到NPC牌，执行NPC效果
  - 可选择使用NPC效果（进入补牌阶段补2张牌）
  - 可选择弃掉NPC，继续翻下一张怪物牌
- C# 源码确认：[XIR.cs:R{N}ZM case, 行 371-406; XIR.cs:R{N}NP case, 行 407-454]

### 阶段4：战斗开始（R{N}Z1）
- **沉默处理**：检查怪物是否具有沉默效果
- **战力初始化**：设置 `Board.InCampaign = true`，启用战力池
- **命中判定**：通过 `PondRefresh` 消息刷新命中状态
- **技能触发**：运行 `R{N}Z1` 阶段的技能（优先级：-400, -300, -100, 300）
- C# 源码确认：[XIR.cs:R{N}Z1 case, 行 456-472]

### 阶段5：战斗开始确认（R{N}Z8）
- 运行 `R{N}Z8` 阶段的技能
- C# 源码确认：[XIR.cs:R{N}Z8 case, 行 473-475]

### 阶段6：出场效果（R{N}CC）
- 读出怪物牌的【出场效果】并执行
- 若出场效果导致触发者死亡，战斗立刻结束，后续流程跳过
- C# 源码确认：[XIR.cs:R{N}CC case, 行 476-489]

### 阶段7：战牌阶段（R{N}PD → R{N}ZC）
- **战牌前**（R{N}PD）：运行技能阶段
- **战牌确认**（R{N}ZC）：启用玩家战力池，刷新命中状态
- **战力计算**：
  - 触发方战力 = 触发者战力（含装备+宠物+技能）+ 支援者战力（若命中）
  - 怪物方战力 = 怪物战力 + 妨碍者战力（若命中）
  - 战力池：`Board.RPool`（触发方）和 `Board.OPool`（怪物方）
- **轮流出战牌**：从战力较低方开始（相等时怪物方先），双方轮流使用战牌
- **每人每场战斗限用1张战牌**（特殊牌不受此限制）
- C# 源码确认：[XIR.cs:R{N}PD case, 行 490-492; XIR.cs:R{N}ZC case, 行 493-499]

### 阶段8：战斗结算（R{N}ZD）
- **胜利判定**：`Board.IsRounderBattleWin()` 返回 true 则触发方胜利
  - 判定逻辑：检查是否有玩家设置 `STRi`（战力锁定），否则比较战力池
  - `CalculateRPool() >= CalculateOPool()` 则胜利
- **胜利效果**：发送 `G1GE,W,怪物ID` 消息执行胜利效果
- **失败效果**：发送 `G1GE,L,怪物ID` 消息执行失败效果
- **宠物获取**：胜利时通过 `HarvestPet` 消息将怪物变为宠物
  - 宠物属性唯一性：已有同属性宠物时，需在两只中选择弃掉一只
- C# 源码确认：[XIR.cs:R{N}ZD case, 行 500-524; XIR.cs:R{N}VS case, 行 533-587]

### 阶段9：战斗结果（R{N}ZN）
- 记录战斗结果：`Board.IsBattleWin` 和 `Board.PoolDelta`
- 关闭战力池和玩家战力池
- C# 源码确认：[XIR.cs:R{N}ZN case, 行 525-532]

### 阶段10：战斗结束准备（R{N}Z2）
- 广播 `G0AX` 消息重置所有玩家的 AX 状态
- C# 源码确认：[XIR.cs:R{N}Z2 case, 行 588-593]

### 阶段11：战斗结束（R{N}ZF）
- 清理战斗状态：重置战力池、回收怪物、清理沉默状态
- 广播 `G0AX` 消息重置所有玩家的 AX 状态
- 发送 `G1ZK,1` 和 `G1HK,1` 消息
- C# 源码确认：[XIR.cs:R{N}ZF case, 行 594-609]

### 阶段12：战斗结束确认（R{N}ZZ）
- 发送 `CoachingSign` 消息标记战斗结束
- C# 源码确认：[XIR.cs:R{N}ZZ case, 行 610-621]

## 四、死亡处理流程

当玩家HP降至0时，按以下顺序处理：

### 步骤1：HP归零检测（G0ZH,0）
- **触发时机**：HP变化后检查是否有玩家HP=0
- **处理逻辑**：
  1. 查找所有HP=0且未使用过倾慕的玩家（`p.IsAlive && p.HP == 0 && !p.Loved`）
  2. 对每个HP=0的玩家，检查其配偶列表（`hero.Spouses` + `player.ExSpouses`）
  3. 根据配偶类型执行不同逻辑：
     - **普通配偶**（数字ID）：找到在场且HP>0的对应角色，自动触发倾慕
     - **慕容紫英类型**（`!1`）：选择一名存活玩家作为倾慕目标
     - **女娲类型**（`!5`）：选择一名女性存活玩家作为倾慕目标
     - **空修类型**（`!6`）：选择一名男性存活玩家作为倾慕目标
     - **白月类型**（`!2`）：若白月宠物在场，自动触发倾慕
     - **世长老类型**（`!9`）：若世长老宠物在场，自动触发倾慕
     - **灵隐类型**（`!3`）：选择所有生物特征为A的存活玩家
     - **玄机类型**（`!4`）：选择所有生物特征为B的存活玩家
     - **七里香类型**（`!7`）：选择所有生物特征为D的存活玩家
     - **魔剑类型**（`!8`）：若魔剑装备在场，自动触发倾慕
  4. 发送 `G0LV` 消息执行倾慕效果
- C# 源码确认：[XIG.cs:G0ZH handler, priority=100, 行 166-341]

### 步骤2：倾慕效果执行（G0LV）
- **处理逻辑**：
  1. 广播倾慕消息
  2. 计算HP变化：倾慕目标获得HP = 倾慕者数量，每个倾慕者损失1HP
  3. 限制HP变化不超过当前HP和HP上限
  4. 通过 `HpIssueSemaphore` 广播HP变化
- **倾慕规则**：
  - 每人每局限1次倾慕机会（`player.Loved = true`）
  - 倾慕是强制的，与队伍无关
  - 倾慕引发的HP损失可能触发链式反应（其他角色的倾慕）
  - 倾慕引发的HP损失**不能用隐蛊免除**（因为不经过伤害处理）
- C# 源码确认：[XIG.cs:G0LV handler, 行 1392-1432]

### 步骤3：死亡检查（G0ZH,200）
- **处理逻辑**：若仍有玩家HP=0，发送 `G0ZW` 消息进入死亡退场流程
- C# 源码确认：[XIG.cs:G0ZH handler, priority=200, 行 335-341]

### 步骤4：死亡退场（G0ZW）
- **优先级0**：广播死亡消息，设置 `Nineteen` 标记
- **优先级100**：
  1. 处理角色退场：设置 `IsAlive = false`, `IsTared = false`, `HP = 0`
  2. 检查队伍存活状态：
     - 若双方均无存活玩家：平局（`G0WN,0`）
     - 若仅一方存活：该方胜利（`G0WN,队伍ID`）
- **优先级200**：
  1. 丢弃手牌（通过 `G0DH,3` 消息丢弃所有手牌）
  2. 丢弃宠物（通过 `LosePet` 消息）
  3. 丢弃救援NPC（通过 `G2OL` 消息）
  4. 丢弃符文（通过 `G0OF` 消息）
- **优先级300**：让队友获得补牌（遗言）
  - 死亡玩家选择一名队友，该队友补2张牌
- **优先级400**：角色最终离场（发送 `G0OY,2,玩家ID` 消息）
- C# 源码确认：[XIG.cs:G0ZW handler, 行 343-435]

### 步骤5：变身检查
- **倾慕复活后也检查变身条件**
- **赵灵儿**：敌方宠物>=3时变身梦蛇（强制）
- **赵灵儿·梦蛇**：敌方宠物<3时还原（强制）
- **魔翳**：队友全灭时变身湮世穹兵（强制）
- **孔璘**：死亡时变身为魔尊（装备/手牌/宠物保留）
- C# 源码确认：[XIG.cs:G0IY handler, 行 1433-1514]

## 五、混战规则

- **触发条件**：每场战斗限1次，翻到怪物牌后可选择再翻1张
- **NPC牌**：触发方战力+NPC战力（若该NPC是触发方任意角色的倾慕者，战力x2）
- **怪物牌**：怪物方战力+新怪物战力；新怪物出场效果和闪避无效
- **双怪物结算**：
  - 胜利：两个怪物都成为宠物
  - 失败：依次执行两个失败效果
- C# 源码确认：[XIR.cs:R{N}ZW case, 行 277-289（放弃战斗时翻怪）]

## 六、横置规则

- **设置横置**：通过 `G0DS,0,1` 消息设置横置状态
  - `player.Immobilized = true`
- **解除横置**：通过 `G0DS,1` 消息解除横置状态
  - `player.Immobilized = false`
- **横置效果**：
  - 被横置的角色下回合跳过除弃牌阶段外的所有阶段
  - 横置不影响当前回合
  - 横置状态下依然可以支援、妨碍、使用非回合限定的手牌
- **解除时机**：弃牌阶段结束后自动解除横置状态
- C# 源码确认：[Player.cs:Immobilized 属性, 行 97; XIR.cs:R{N}00 case, 行 125-136; XIG.cs:G0DS handler, 行 1516-1532]

## 八、胜利条件

满足任一即结束：
1. **歼灭胜利**：敌方全部角色死亡退场
   - 通过 `G0WN,队伍ID` 消息触发
2. **宠物战力胜利**：怪物牌堆耗尽时，计算双方宠物战力之和
   - 通过 `G1WJ` 消息触发，计算 `CalculatePetsScore()`
   - `Board.FinalAkaScore > Board.FinalAoScore` 则A队胜利
3. **认输**：敌方玩家认为无法取胜，中途认输
- C# 源码确认：[XIG.cs:G0WN handler, 行 2572-2592; XIG.cs:G1WJ handler, 行 716-730]

## 九、1对1游戏特别规则

- 每名玩家选择两个游戏角色
- 两名角色共用手牌
- 两名角色HP、装备、宠物不共用
- 可以任意使用两名角色的技能
- 触发战斗时，可以任意选择哪个角色触发，哪个角色支援
- 作为妨碍方时，可以任选一名角色妨碍
- 角色被横置时，两名角色同时横置
- 只要有一个是慕容紫英，弃牌上限即为5张
- 2对2游戏中也可启用此规则（合计8名角色），但触发战斗时不能用自己的另一名角色支援

## 十、其他规则

### 双方队伍人数不同
若双方队伍起始人数不一致，请让其中一名玩家同时控制两名角色，手牌分开使用。

### 争人法（替代选角方式）
展示12张角色牌，A、B双方轮流从中剔除一张，然后按以下顺序选择：
1. A队选1名
2. B队选2名
3. A队选2名
4. B队选1名
双方选够后自行分派各玩家使用的角色。
