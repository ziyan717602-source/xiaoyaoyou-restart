## 背景

Phase 5 聚焦于怪物/NPC 效果引擎，而事件牌（Evenement）作为独立牌类在 C# 原版中有独立的效果系统（`EveCottage` 类），当前项目遗漏了事件牌效果的实现。

**C# 原版参考**：`reference/psd48-master/PSDGamepkg/JNS/SJ101.cs`（EveCottage 类，所有事件效果在此注册）

**SPI 位掩码说明**（来源：`Evenement.cs`）：
- `H` = HP伤害事件（IsHarmInvolved = true，位 0x1）
- `T` = 手牌效果事件（IsTuxInvolved = true，位 0x2）
- `T#` = 手牌效果仅针对自己（位 0x4）
- `S` = 沉默事件（IsSilence = true，位 0x8）
- `TH` = 同时涉及HP和手牌

## 关键设计决策

### 事件效果 vs 技能效果

事件效果与技能效果的区别：
- **事件效果**由翻取事件牌触发，效果作用于全场或指定玩家
- **技能效果**由角色技能触发，有明确的触发时机（OCCURS）和优先级
- 事件效果不走技能系统（SkillResolver），有独立的解析器

### SPI 位掩码处理

```typescript
// 位掩码定义（与 C# Evenement.cs 一致）
const SPI_HARM = 0x1;   // H - HP伤害事件
const SPI_TUX = 0x2;    // T - 手牌效果事件
const SPI_TUX_SELF = 0x4; // T# - 仅针对自己
const SPI_SILENCE = 0x8; // S - 沉默事件
```

### 玩家选择处理

需要玩家选择的事件（如 SJ103 指定目标、SJ104 轮流选牌）通过 `pendingChoices` 机制处理：
- 设置 `pendingChoice` 等待玩家响应
- 玩家通过 move 提交选择
- 事件效果根据选择继续执行
